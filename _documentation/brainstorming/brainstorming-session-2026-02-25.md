### Phase 2: First Principles Thinking

Stripped all assumptions. Rebuilt from three fundamental truths:

**Fundamental Truth #1:** Governance requires visibility of three things: what each agent CAN do, what it is FORBIDDEN from doing, what happens when it is UNCERTAIN. These must be expressed as explicit typed constraints — not comments.

**Fundamental Truth #2:** Traceability means a hiring manager can run the demo and see — in the CLI — exactly what was classified, who handled it, what was done, what was returned with what confidence. No LangSmith required. Structured `[DISPATCHER] intent=X confidence=0.87 → routing to ResearchAgent` IS observability in MVP form.

**Fundamental Truth #3:** RAG is not about the vector database. RAG is about knowing WHEN to retrieve from structured knowledge vs WHEN to search the web vs WHEN to query memory. The dispatcher making that three-way routing decision is the intelligent behavior worth demonstrating.

**Key Decisions from First Principles:**
- Memory and Guardrails are NOT separate agents — they are cross-cutting services wired into the graph
- File-based memory is a feature, not a compromise — it makes the memory model inspectable by the hiring manager mid-demo
- A clearly-labeled `BookingAgent` stub is more honest and impressive than a fake booking flow

### Phase 3: SCAMPER

7 lenses applied to the MVP skeleton. 8 concrete ideas generated.

**S (Substitute):** Hybrid Dispatcher — rule-based pre-filter + LLM for ambiguous cases. Shows cost-awareness and determinism preference.

**C (Combine):** Blended Response with Source Attribution — Research Agent + RAG Agent results merged with source labels before response.

**A (Adapt from TNL-HELP):** Prompt Orchestration Layer + LangGraph Flow Visibility — both adapted directly from TNL-HELP production patterns.

**M (Modify):** Proactive Concierge Opening — memory-driven greeting before user says anything. "Welcome back, Andrey. Based on your trip to Bali last March..."

**P (Put to other uses):** Memory folder as demo narrative tool — pre-seeded "Alex" persona makes personalization visible from the first run.

**E (Eliminate):** Booking flow cut entirely — BookingAgent stub with integration contract comment is more honest and saves significant build time.

**R (Reverse):** Agent-Initiated Follow-Up — after Research Agent returns results, a conditional proactive_suggestion_node injects a follow-up. The job description's "proactive outcome completion" expressed as a LangGraph edge.

## Idea Organization and Prioritization

### All Ideas by Theme

**Theme 1: Architecture & Graph Design**
- **Dispatcher-First Pattern:** Client → Dispatcher → [RAG | Research | Fallback] → Guardrail → Response. The proof of architectural thinking.
- **Hybrid Dispatcher with Confidence Ladder:** Rule-based pre-filter for obvious intents, LLM escalation for ambiguous. Confidence score on every routing decision.
- **LangGraph Flow Visibility in CLI:** Every graph traversal printed: `[FLOW] intent_clarification → rag_lookup → response_synthesis → guardrail_check`
- **BookingAgent Honest Stub:** Stub with `# In production: connect to booking API` comment. Shows boundary awareness.
- **Agent-Initiated Follow-Up Node:** Conditional `proactive_suggestion_node` after research completion — graph edge as business feature.

**Theme 2: Knowledge & Retrieval**
- **Mock-But-Honest Knowledge Layer:** Local JSON KB (destinations, prices, hotels) exposing same interface as Bedrock Knowledge Bases. Production swap point labeled.
- **Five-Mode Routing Logic:** The dispatcher's three-way routing decision (cached KB vs live web vs memory) IS the intelligent behavior.
- **Blended Response with Source Attribution:** Internal KB + web search merged, sources cited: "Based on our current offerings [RAG] and latest travel trends [Web]..."

**Theme 3: Memory & Personalization**
- **File-Based Memory as a Feature:** `memory/sessions/{session_id}/` and `memory/profiles/{user_id}/` — transparent, inspectable, git-committable.
- **Pre-seeded "Alex" Persona:** Demo character with 2 past trips, 1 cached research session. Personalization visible from first run.
- **Proactive Concierge Opening:** Session start loads profile → memory-driven greeting. Proves "proactive outcome completion" in 5 seconds.

**Theme 4: Methodology & Presentation**
- **Methodology IS the MVP:** Spec → architecture → implementation → runnable demo. The narrative arc of the repo.
- **Repo as Candidate Profile:** Every file answers a job description bullet. Business case framing, BMAD docs, SDD principles.
- **Prompt Orchestration Layer:** Prompts in `prompts/{agent_name}/{version}.yaml` — spec-driven, versioned, swappable without touching graph logic.
